# Week4-01: IPv6 基本設定 + ACL

## トポロジー

```
[R1] Fa0/0 ─── Fa0/0 [R2] Fa0/1 ─── Fa0/0 [R3]
 2001:db8:1::1/64      2001:db8:1::2/64
                       2001:db8:2::1/64      2001:db8:2::2/64
```

## IPアドレス

| 機器 | インタフェース | アドレス |
|------|-------------|---------|
| R1 | Fa0/0 | 2001:db8:1::1/64 |
| R2 | Fa0/0 | 2001:db8:1::2/64 |
| R2 | Fa0/1 | 2001:db8:2::1/64 |
| R3 | Fa0/0 | 2001:db8:2::2/64 |

## 要件

### R1（ポート 5002）

ターミナル環境設定:
- ホスト名: R1
- 特権パスワード: cisco
- DNSルックアップ無効化
- コンソール: ログ同期・タイムアウト無効
- タイムゾーン: JST +9
- ターミナル行数: 無制限

IPv6:
- IPv6ルーティングを有効化すること
- Fa0/0 に IPv6 アドレスを設定すること
- R3 のネットワーク（2001:db8:2::/64）へのスタティックルートを R2 向けに設定すること

### R2（ポート 5003）

ターミナル環境設定: R1と同様（ホスト名: R2）

IPv6:
- IPv6ルーティングを有効化すること
- Fa0/0・Fa0/1 に IPv6 アドレスを設定すること

ACL:
- R3 から R1 への通信を拒否する IPv6 ACL を設定すること
- ACL名: BLOCK_R3
- ND（Neighbor Discovery）は許可すること
- Fa0/1 の inbound に適用すること

ACL構成:
```
deny ipv6 2001:db8:2::/64 host 2001:db8:1::1
permit icmp any any nd-na
permit icmp any any nd-ns
permit ipv6 any any
```

### R3（ポート 5005）

ターミナル環境設定: R1と同様（ホスト名: R3）

IPv6:
- IPv6ルーティングを有効化すること
- Fa0/0 に IPv6 アドレスを設定すること
- R1 のネットワーク（2001:db8:1::/64）へのスタティックルートを R2 向けに設定すること

## テスト

| 送信元 | 宛先 | 結果 |
|--------|------|------|
| R3 | 2001:db8:1::1 (R1) | 通らない（ACLでブロック） |
| R3 | 2001:db8:1::2 (R2) | 通る |

## 注意点

- ACLのdenyは`permit ipv6 any any`より前に書くこと（順序が重要）
- ACLをFa0/0ではなくFa0/1に適用すること（R3側のインタフェース）
- NDを許可しないとIPv6隣接解決が壊れる
- IPv4と違い`ipv6 traffic-filter`で適用する
