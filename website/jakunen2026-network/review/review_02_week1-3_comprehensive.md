[R-ISP]                         [R-CE] ← あなたが設定するルータ

## 要件

### 1. ターミナル環境設定（R-CE のみ）
標準のターミナル環境設定を行う（暗記で投入）。

### 2. インタフェース設定（R-CE）
- Fa0/0: 203.0.113.2/30、no shutdown
- Lo0: 192.168.10.1/32、no shutdown

### 3. スタティックルーティング（R-CE）
- デフォルトルート → 203.0.113.1（R-ISP）

### 4. 静的NAT（R-CE）
- Lo0のIPアドレス `192.168.10.1` を グローバルアドレス `203.0.113.6` に静的変換する
- Fa0/0: `ip nat outside`
- Lo0: `ip nat inside`

### 5. 動的PAT（R-CE）
- 送信元アドレス `192.168.10.0/24` を Fa0/0 に overload する
- 標準ACLを使って対象ネットワークを定義すること

### 6. 拡張ACL（R-CE）
- Fa0/0 の **inbound** に適用する
- 許可: established TCP（内部からの通信の戻りパケット）
- 許可: ICMP echo-reply（内部からのpingの応答）
- 拒否: それ以外すべて

## 採点基準（達成確認）

| # | 確認内容 | コマンド |
|---|---------|---------|
| 1 | R-CE の全IFがup/up | `show ip interface brief` |
| 2 | R-CE から R-ISP Fa0/0 に ping が通る | `ping 203.0.113.1` |
| 3 | R-CE から 8.8.8.8 に ping が通る（source Lo0） | `ping 8.8.8.8 source Lo0` |
| 4 | 静的NAT変換エントリが存在する | `show ip nat translations` |
| 5 | PAT変換エントリが生成される | `show ip nat translations`（ping後） |
| 6 | R-ISP から 203.0.113.2 への ping が **ブロック** される | R-ISPで `ping 203.0.113.2` → fail |
| 7 | ACLが正しく適用されている | `show ip access-lists` |
