# 課題5: 静的NAT

## トポロジー

```
          内部ネットワーク          外部ネットワーク
  192.168.1.0/24                    203.0.113.0/30
  [PC]          Fa0/1   Fa0/0                  [R2/ISP]
  .10    ----   [R1]  .1 ---- .2   203.0.113.1
  192.168.1.10  .1
```

| ノード | IF | IPアドレス | 役割 |
|--------|-----|-----------|------|
| R1 | Fa0/0 | 203.0.113.1/30 | 外部（ISP向け）|
| R1 | Fa0/1 | 192.168.1.1/24 | 内部LAN |
| R1 | Lo0 | 10.10.10.1/32 | PC代わり |
| R2 | Fa0/0 | 203.0.113.2/30 | ISP側 |

※PCはR1のFa0/1に繋がっている想定。今回はR1のLoopback0（10.10.10.1/32）でPC代わりに使う。
※192.168.1.10はFa0/1のサブネット（192.168.1.0/24）と重複するため使用不可。

## 事前準備
- R1・R2を `write erase` + `reload`
- ターミナル環境設定（暗記で）
- R1にデフォルトルート → R2（203.0.113.2）
- R2にスタティックルート → 203.0.113.0/30（直接接続）

## 要件

### Step 1: インタフェース設定（R1・R2）
上記IPアドレスを設定し、up状態にする。

### Step 2: Loopback設定（R1のみ）
R1に `Loopback0` を作成し `192.168.1.10/32` を割り当てる。（PC代わり）

### Step 3: ルーティング
- R1: デフォルトルートをR2向けに設定
- R2: 特に不要（直接接続のみ）

### Step 4: 静的NAT設定（R1）
- 内部アドレス `10.10.10.1` を外部アドレス `203.0.113.10` に静的に変換する
- Fa0/1を内部（inside）、Fa0/0を外部（outside）として設定する

### Step 5: R2にスタティックルート追加
- `203.0.113.10` は `203.0.113.0/30` の範囲外のため、R2から到達するにはルートが必要
- `203.0.113.10/32` → R1（203.0.113.1）向けにスタティックルートを設定する

## 達成確認
- R2から `ping 203.0.113.10` が通ること
- R1で `show ip nat translations` に変換エントリが表示されること

## ポイント
- NATには inside / outside の方向設定が必要
- 静的NATは1対1の固定変換
- `show ip nat translations` で変換テーブルを確認できる
