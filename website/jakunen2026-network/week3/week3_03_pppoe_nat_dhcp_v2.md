# Week3-03: PPPoE + 動的PAT + DHCP（バリアント2）

## トポロジー

```
[R-WS] Fa0/0 ─── Fa0/1 [R-BR] Fa0/0 ─── Fa0/0 [R-WAN]
  DHCPクライアント       GW/DHCP/PAT        PPPoEサーバ（設定済み）
172.20.0.x/24        172.20.0.1/24                198.51.100.1/24(Lo0)
                     Dialer0: PPPoEで自動取得
```

## IPアドレス

| 機器 | インタフェース | アドレス |
|------|-------------|---------|
| R-WAN | Loopback0 | 198.51.100.1/24 |
| R-BR | Dialer0 | PPPoEで自動取得 |
| R-BR | Fa0/1 | 172.20.0.1/24 |
| R-WS | Fa0/0 | DHCPで自動取得 |

## 要件

### R-BR（ポート 5001）

ターミナル環境設定:
- ホスト名: R-BR
- 特権パスワード: cisco
- DNSルックアップ無効化
- コンソール: ログ同期・タイムアウト無効
- タイムゾーン: JST +9
- ターミナル行数: 無制限

PPPoEクライアント:
- R-WAN との間に PPPoE 接続を確立すること
- 認証方式: CHAP（username: BRANCH / password: netlab）
- MTU: 1492
- デフォルトルートを Dialer0 向けに設定すること

LAN インタフェース:
- Fa0/1 に 172.20.0.1/24 を設定すること

DHCPサーバ:
- プール名: BRANCH_POOL
- ネットワーク: 172.20.0.0/24
- デフォルトゲートウェイ: 172.20.0.1
- DNSサーバ: 198.51.100.1
- 除外アドレス: 172.20.0.1 〜 172.20.0.20

動的PAT:
- 172.20.0.0/24 からインターネットへの動的 PAT を設定すること
- NAT outside は Dialer0

その他:
- `no ip cef`（GNS3環境の制約）

### R-WS（ポート 5004）

ターミナル環境設定:
- ホスト名: R-WS
- 特権パスワード: cisco
- DNSルックアップ無効化
- コンソール: ログ同期・タイムアウト無効
- タイムゾーン: JST +9
- ターミナル行数: 無制限

インタフェース:
- Fa0/0: ip address dhcp / no shutdown
