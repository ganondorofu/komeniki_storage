# 課題3: スタティックルーティング（2台間）

## トポロジー

```
[R1] Fa0/0 -------- Fa0/0 [R2]
      10.0.0.1/30   10.0.0.2/30

R1 Fa0/1: 192.168.1.1/24  (LAN側)
R2 Fa0/1: 192.168.2.1/24  (LAN側)
```

## 事前準備
- R1、R2 それぞれ `write erase` + `reload`
- ターミナル環境設定を暗記で実施してから本課題へ進む

### ターミナル環境設定 要件（毎回共通）
1. DNS検索を無効化
2. タイムゾーンを JST（UTC+9）に設定
3. ホスト名を設定
4. enableパスワードを暗号化方式で `cisco`
5. コンソール: 自動ログアウト無効
6. コンソール: More表示無効
7. コンソール: ログ割り込み後に入力文字を補完
8. コンソール: 常に特権モードでアクセス可能
9. コンソールパスワード `cisco`、ログイン有効
10. VTY 0〜4 パスワード `cisco`、ログイン有効

## 要件

### 共通（R1・R2）
1. ターミナル環境設定（毎回暗記で実施）
2. 各インタフェースにIPアドレスを設定し、up状態にする

### R1
3. R2のLAN（192.168.2.0/24）へのスタティックルートを設定する
   - ネクストホップ: 10.0.0.2

### R2
4. R1のLAN（192.168.1.0/24）へのスタティックルートを設定する
   - ネクストホップ: 10.0.0.1

## 達成確認
- R1から `ping 192.168.2.1` が通ること
- R2から `ping 192.168.1.1` が通ること
- `show ip route` でスタティックルートが `S` で表示されること

## ポイント
- `ip route [宛先NW] [サブネットマスク] [ネクストホップIP]`
- ルーティングは「どこに行くか」を教えるだけ。戻り経路も忘れずに
